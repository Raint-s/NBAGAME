# 项目运行文档体系落地计划

日期：2026-06-02

## 目标

建立一套公开可读、便于持续维护的项目文档机制，让仓库能够回答：

- 当前项目到哪一步了
- 最近完成了什么
- 现在有哪些已知风险
- 新工作开始前的计划放在哪里

## 范围

- 增加项目状态总览
- 增加 changelog
- 增加首份审计归档
- 增加计划目录与模板
- 将这套规则写入现有公开文档

## 本次不做

- 不在这一步直接重构游戏代码
- 不在这一步完成目录第二阶段整理
- 不在这一步修复全部审计问题

## 风险

- 如果只写文档不写使用规则，后续容易再次失效
- 如果文档太重，会降低团队执行意愿

6. [x] 将规则写入 README 与 DEVELOPMENT

## 完成标准

- 文档入口清楚
- 当前阶段状态有明确落点
- 后续每轮工作都知道该更新哪些文档
